# 用户故事（User Stories）

## 1. 公众号作者场景

### 用户信息
- **姓名**：李明
- **角色**：公众号作者（科技领域）
- **写作频率**：每周2-3篇长文（1500-2000字）
- **痛点**：
  - 写作速度慢，常常熬夜赶稿
  - AI写作缺乏深度和个人观点
  - 逻辑结构混乱，需要反复修改

### 目标
快速生成结构清晰、有深度的科技领域公众号文章，同时保持个人风格和观点。

### 使用场景
1. **构思阶段**：
   - 在思维导图中创建中心主题「AI发展对未来工作的影响」
   - 添加多个分支：「自动化趋势」、「技能转型」、「行业变革」等
   - 为每个分支添加具体论点和数据支持

2. **资料收集**：
   - 在相关节点添加参考资料（行业报告链接、专家观点）
   - 开启「基于资料写作」模式

3. **生成文章**：
   - 先生成整体框架，确保逻辑流畅
   - 锁定已确认的关键段落
   - 对需要加强的部分进行局部重写

4. **编辑与导出**：
   - 对AI生成的内容进行微调
   - 导出为Markdown格式，直接用于公众号排版

### 期望结果
- 从构思到成稿时间缩短50%
- 文章结构清晰，逻辑严密
- 包含引用来源，提升可信度
