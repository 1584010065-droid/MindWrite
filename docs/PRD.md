# PRD：思维导图 + AI 实时写作产品

版本：v0.3（交付版草案）
更新时间：2026-03-14（Asia/Shanghai）

## 0. 术语与范围

术语定义（用于统一交付口径）：

- `节点（Node）`：思维导图中的一个条目，承载观点/论点/素材。
- `文段（Block）`：文章预览区中的一个可独立编辑单元，默认对应一个节点，可配置为一个节点对应多个文段。
- `映射（Binding）`：`Node <-> Block` 的可追溯关系，用于“只改影响范围”和“结构可视化”。
- `锁定（Lock）`：防止 AI 覆盖用户已确认的文段内容。
- `基于资料写作（Grounded）`：生成时优先依据节点资料（URL/粘贴原文）进行写作，输出带引用来源信息。
- `增量生成（Incremental）`：只生成或重写“选中的节点/文段”对应的输出，不重排、不污染其它内容。

产品范围（MVP）：

- 单人写作场景为主，不做协作。
- 以中文长文为主（公众号/博客），风格先固定，预留扩展点。
- 存储先本地（LocalStorage/IndexedDB），后续再接云端。

非目标（MVP 明确不做）：

- 历史版本与多分支草稿管理。
- 多模型选择、复杂模板市场。
- 付费墙后的深度内容抓取与网页全文解析（MVP 不做内置浏览器抓取）。

## 1. 产品概述

### 1.1 产品定位

一款结构化写作工具：用户用思维导图输入核心观点与逻辑链路，AI 实时把结构扩写为可发布的长文。解决“AI 写作太浅、自己写太慢”的痛点。

### 1.2 核心价值

- 你控制思想深度：导图承载观点、论据与逻辑链路（可追溯、可控重写边界）。
- AI 提升表达效率：自动完成段落扩写、过渡、例证、语气一致与排版输出。

### 1.3 目标用户

- 公众号作者
- 博客创作者
- 自媒体人/KOL
- 需要长文写作的内容创作者

## 2. 目标与成功标准

### 2.1 产品目标（MVP）

1. 用户能用导图快速搭结构，并将节点逐段扩写为成文。
2. 用户能反复重写但保持“影响边界可控”：只改选中范围，锁定内容不被覆盖。
3. 用户能引入资料作为证据输入，降低幻觉，提高“写得深”的概率。

### 2.2 成功指标（MVP 后 4 周观察口径）

| 指标 | 目标 |
|-----|-----|
| 30 日留存 | > 15% |
| DAU | > 100 |
| 免费 -> 付费 | > 5% |
| NPS | > 40 |
| Time-to-first-draft | 从导图到首稿中位数 < 10 分钟 |
| 可控性指标 | “全文重写后锁定块未被改写”成功率 > 99% |

## 5. 关键交互流程（可验收）

### 5.1 流程 A：分段生成（增量生成）

1. 用户在导图中选中 1 个或多个节点。
2. 点击「生成选中段落」。
3. 系统展示影响范围：将生成/重写哪些 blocks（按映射规则）。
4. 系统生成完成后：仅新增/更新选中节点对应 blocks；锁定 blocks 不变；未选中节点对应 blocks 不变。
5. 用户可对 blocks 进行编辑/锁定/局部重写。

验收要点：

- 未选中节点对应 blocks 的 `content` 与顺序不变（除非用户手动调整）。
- 同一节点重复生成时，默认行为是“覆盖该节点的未锁定且未被用户编辑的 blocks”；若该节点对应 blocks 存在 `isUserEdited=true`，则必须二次确认才允许覆盖（避免误操作）。

### 5.2 流程 B：全文重写（保留锁定）

1. 用户点击「生成全文」。
2. 系统展示影响范围：将重写哪些 blocks（所有未锁定 blocks）。
3. 生成完成后：锁定 blocks 原文保留；其它 blocks 按导图结构重写；block 顺序不变。

验收要点：

- 锁定 blocks 字符串完全一致（MVP 采用“完全锁定”语义，避免歧义）。

### 5.3 流程 C：局部重写（block 级）

1. 用户在右侧选中一个 block，点击「重写/扩展/缩短」。
2. 系统只重写该 block（或该 block 的子片段，若支持选中文字）。
3. 其它 blocks 不得被改写或重排。

## 6. 详细需求（P0）

### 6.1 思维导图编辑器（P0）

功能点：

- 创建节点：Enter 新增同级；Tab 新增子级（可调整）。
- 编辑节点：双击或快捷键进入编辑；Esc 取消。
- 格式编辑：支持文字粗体、斜体、颜色、下划线等基本格式。
- 拖拽重排：拖拽节点改变父子关系与同级顺序。
- 折叠/展开：控制节点子树显示。
- 多选：支持多选节点并一次生成多个段落。

验收标准：

- 节点的 `id` 稳定，重排不改变 `id`。
- 节点内容为空时不允许触发生成（提示先补充内容或删除节点）。
- 格式编辑功能正常工作，且在导出时正确渲染。

### 6.2 节点参考资料（P0）

功能点：

- 每个节点可添加 0..N 条资料：类型为 `url` 或 `text`（粘贴原文片段）。
- 资料在 UI 中可编辑、删除、排序；支持一键“复制引用”。
- 生成时可勾选“基于资料写作（grounded）”，默认规则：当选中节点存在资料时自动开启。

验收标准：

- grounded 开启时，生成输出包含引用列表（最小实现：列出资料标题/URL；不要求严谨学术格式）。
- grounded 关闭时，不输出引用列表。

### 6.3 节点-文段映射（P0）

映射规则（MVP 采用最简单可控版本）：

1. 默认：一个节点对应一个 block。
2. 当节点内容较长或用户选择“拆分段落”时，可允许一个节点对应多个 blocks（P1 再做 UI；P0 先保留数据结构支持）。
3. blocks 的展示顺序严格等于导图的 DFS 顺序（从中心主题开始，先父后子，同级按用户顺序）。
4. 节点移动后：对应 blocks 跟随节点在文章中的位置移动；block 内容不变。
5. 节点删除后：提示用户选择“删除对应 blocks”或“保留为未绑定 blocks（orphan）”（MVP 默认删除）。

验收标准：

- 点击某个 block 时，对应节点高亮；点击节点时，对应 block 滚动定位并高亮。
- 任何 AI 操作必须声明其影响的 `nodeIds` 或 `blockIds`，并严格限制在范围内。

### 6.4 锁定机制（P0）

锁定语义（MVP）：

- 锁定为“完全锁定”：AI 不得修改该 block 的任何字符。
- 锁定块参与上下文：锁定内容可用于帮助 AI 生成相邻段落的衔接，但输出时不得覆盖锁定文本。

验收标准：

- 分段生成、全文重写、局部重写均不得改动锁定 block。

### 6.5 分段生成（P0）

输入：

- 选中节点集合 `nodeIds`。
- 当前导图结构（节点树）。
- 已存在 blocks（含锁定标记、绑定关系）。
- grounded 开关与选中节点的资料内容（若启用）。

输出（必须）：

- 新增或更新的 blocks（含绑定的 nodeId）。
- 引用列表（仅 grounded 模式）。
- 本次影响范围（nodeIds/blockIds）。

验收标准：

- 未在影响范围内的 blocks 字符串完全不变（防回归）。

### 6.6 文章预览与编辑（P0）

功能点：

- 右侧以 blocks 列表呈现，支持编辑、撤销/重做（最小可用可只做浏览器级撤销）。
- 格式编辑：支持文字粗体、斜体、颜色、下划线等基本格式。
- block 工具条：锁定/解锁、重写（局部）、删除（删除需同步解除映射或删除节点，MVP 先仅允许从导图侧删除）。
- 引用展示：grounded 模式下在 block 底部展示"引用来源"。

验收标准：

- 用户手动编辑后的 block 标记为"已编辑"，默认不会被分段生成覆盖，除非用户明确选择"覆盖该节点段落"（二次确认）。
- 格式编辑功能正常工作，且在导出时正确渲染。

## 7. 数据结构与状态（建议实现）

```typescript
type Id = string;

type NodeSource =
  | { id: Id; type: "url"; value: string; title?: string }
  | { id: Id; type: "text"; value: string; title?: string };

type TextStyle = {
  bold?: boolean;
  italic?: boolean;
  underline?: boolean;
  color?: string;
};

type TextSegment = {
  text: string;
  style?: TextStyle;
};

interface MindMapNode {
  id: Id;
  content: TextSegment[];  // 支持格式化的文本内容
  sources?: NodeSource[];
  children: MindMapNode[];
  isExpanded: boolean;
}

type LockMode = "none" | "full";

interface Citation {
  sourceId: Id;      // NodeSource.id
  title?: string;
  url?: string;
  quote?: string;    // 可选：短摘录（避免长引用）
}

interface ArticleBlock {
  id: Id;
  content: TextSegment[];  // 支持格式化的文本内容
  sourceNodeId: Id;  // 绑定节点
  lockMode: LockMode;
  isUserEdited?: boolean;
  citations?: Citation[];
  lastGeneratedAt?: string; // ISO
}

interface Article {
  id: Id;
  title: string;
  blocks: ArticleBlock[];
  createdAt: string; // ISO
  updatedAt: string; // ISO
}
```

## 8. 非功能需求（NFR）

- 性能：导图 300 节点以内编辑流畅；文章区 300 blocks 不明显卡顿。
- 生成时延：分段生成 5-20s 可接受，超时给出重试与缩小范围建议。
- 稳定性：生成失败不丢失用户编辑内容；所有 AI 调用可重试且幂等（同一 requestId 重试不产生重复 blocks）。
- 隐私：MVP 默认本地存储；用户显式点击生成时才发送必要上下文到模型；资料内容属于敏感输入，需明确提示。
- 合规与版权：grounded 模式下引用展示采用“短摘录 + 链接/标题”，避免长段复制粘贴。

## 9. 商业模式与额度（与成本控制对齐）

MVP 定价不强制上线，但 PRD 需要与工程实现对齐：

- 免费版：每日固定点数额度，同时埋点记录实际 token 消耗，为后续点数定价做基线。
- 付费版：更高点数额度 + 导出功能。
- 专业版：更高点数额度 + 导入 + 高级风格。

额度模型（基于点数的计费）：

- 点数与现金关系：1 人民币 = 10 点数。
- 点数消耗规则：不同 AI 动作消耗不同点数（具体计费标准后续确定）。
- 点数与 token 关系：点数消耗与 token 消耗相关，但具体换算比例待定。
- UI 在执行前展示预计点数消耗与剩余额度（减少不确定性与成本失控）。
- 预留点数购买与管理入口，支持用户查看点数余额、消耗记录和购买点数。

## 15. AI 上下文与提示词策略（工程约束）

目标：在保证“影响边界严格”的前提下，尽量降低 token 成本并提升一致性。

原则（MVP 必须实现）：

1. 显式范围：每次请求必须携带 `requestId`、影响的 `nodeIds` 或 `blockIds`，并在模型输出后做校验，禁止越权改写。
2. 结构优先：上下文始终包含导图结构（可用压缩格式），而不是整篇全文原文。
3. 相邻摘要：除目标 blocks 外，只提供相邻 blocks 的短摘要（例如上一段/下一段各 1-3 句），用于衔接与语气一致。
4. 锁定块只读：锁定 blocks 作为只读上下文提供，但输出时不得覆盖，且需要在后处理阶段做字符串一致性校验。
5. grounded 最小化：grounded 开启时只喂入“节点资料的必要片段”，并限制引用摘录长度，避免大段搬运。

验收标准：

- 任何一次 AI 操作都能在日志/埋点中复现：输入范围、grounded 开关、上下文摘要大小、输出影响范围。

## 16. 技术实现建议（非强制）

技术选型（vibe coding 友好）：

- 前端：React + TypeScript
- UI：Tailwind CSS + shadcn/ui
- 导图/图编辑：react-flow（优先）或同类节点编辑方案
- 状态管理：Zustand
- AI 接入：OpenAI API / Claude API（MVP 单一模型）
- 部署：Vercel
- 存储：LocalStorage（MVP）→ Supabase（后期）

存储建议：

- 文章与导图保存为单一文档对象（`Article` + 根 `MindMapNode`），本地定期自动保存。
- 生成请求与响应保存 `requestId` 便于追踪与重试。

## 17. 交付与验收清单（MVP）

交付物：

- 可用的导图编辑器与文章预览区（同屏）。
- 分段生成、局部重写、全文重写三类 AI 动作可用，且影响边界可验证。
- 锁定机制生效（任何 AI 动作不改锁定 block）。
- 节点资料与 grounded 输出引用展示可用。
- 导出 Markdown/纯文本可用。

验收用例（最小集合）：

1. 选中节点 A 生成段落，节点 B 的段落不变。
2. 锁定段落后执行“生成全文”，锁定段落字符串完全一致。
3. 对某段执行“扩展”，只影响该段，邻段不变且衔接自然。
4. 开启 grounded：输出包含引用来源；关闭 grounded：不包含引用来源。
